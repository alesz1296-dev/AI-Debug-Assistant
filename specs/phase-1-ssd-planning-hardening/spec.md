# Phase 1 Spec: SSD Planning Hardening

## Goal

Convert the project roadmap into explicit SSD phases with verifiable gates before implementation expands.

## Requirements

- Add a phase map that defines the full implementation order.
- Ensure every phase has `spec.md`, `plan.md`, and `tasks.md`.
- Separate public portfolio documentation from ignored private guidance notes.
- Record that the project is not deployment-ready or cloud-ready today.

## Acceptance Criteria

- `specs/README.md` exists and defines the end goal, phase order, and development mode transition.
- Every planned phase has requirements, acceptance criteria, validation notes, and out-of-scope boundaries.
- Private daily/manual notes are ignored by git.
- Public docs describe the current state without overstating readiness.

## Out Of Scope

- Application code changes.
- Persistence implementation.
- Cloud infrastructure.
